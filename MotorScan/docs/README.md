# MotorScan

## 專案目標

本專案旨在建立一個馬達特性分析工具。系統核心為一顆 ESP32 開發板，它提供一個 WiFi 網頁介面，讓使用者能啟動一個自動化掃描程序。此程序會逐步增加施加於馬達的 PWM 驅動力，並同步量測其轉速 (RPM)。掃描完成後，系統會在網頁上呈現 PWM 與 RPM 的對應關係圖表。

這個工具的主要目的是幫助開發者快速掌握特定馬達的物理特性，例如：
*   **啟動區 (Dead Zone)**: 找到能讓馬達開始轉動的最低 PWM 值。
*   **線性區**: 觀察馬達在哪個 PWM 區間內，其轉速與 PWM 值大致成正比。
*   **飽和區**: 確定馬達達到最高轉速時的 PWM 值。

這些數據對於後續設計精準的 PID 控制器或進行馬達響應線性化補償至關重要。

## 系統架構

1.  **使用者介面 (MotorWebUI)**:
    *   ESP32 作為 Web Server，提供一個操作介面與結果顯示頁面。
    *   使用者可以透過網頁上的按鈕來「啟動/停止」馬達掃描。
    *   掃描過程中，網頁會即時接收後端傳來的 `(PWM, RPM)` 數據點。
    *   掃描結束後，網頁前端會使用圖表庫（例如 Chart.js）將所有數據點繪製成 PWM-RPM 關係圖，讓使用者可以直觀地分析馬達特性。
    *   通訊方式為 WebSocket。

2.  **馬達掃描器 (MotorScanner)**:
    *   接收來自網頁的掃描指令。
    *   執行一個自動化測試序列：從 PWM `0` 開始，以固定的步進（例如 `+5`）增加 PWM 值，直到達到最大值 `255`。
    *   在每個 PWM 階級，它會維持一小段時間讓馬達速度穩定，然後讀取編碼器的回饋值，計算出目前的 RPM。
    *   將每一組 `(PWM, RPM)` 數據透過 WebSocket 傳送給前端網頁。

3.  **馬達介面 (IMotor)**:
    *   與 MotionPlanner 專案共用相同的馬達抽象介面，統一了 `drive()` (設定 PWM) 與 `getCurrRPM()` (取得轉速) 等操作。
    *   這種設計讓 `MotorScanner` 可以無縫地與真實馬達 (`Motor`) 或模擬馬達 (`MotorSim`) 協同工作，方便在沒有硬體時也能進行軟體開發與測試。

## 核心元件

*   **硬體**: ESP32 開發板
*   **開發環境**: PlatformIO
*   **框架**: Arduino
*   **主要程式庫**:
    *   `ESPAsyncWebServer`: 提供高效能的非同步網頁伺服器功能。
    *   `AsyncTCP`: `ESPAsyncWebServer` 的底層依賴。
    *   `WebSockets`: 用於實現 ESP32 與網頁前端之間的即時雙向通訊。

## 如何開始

1.  **硬體連接**:
    *   根據 `src/main.cpp` 中定義的腳位，將 ESP32 與您的馬達驅動板、馬達編碼器連接。

2.  **環境設定**:
    *   安裝 Visual Studio Code 與 PlatformIO 擴充功能。
    *   開啟此專案資料夾。

3.  **軟體設定**:
    *   將 `src/config.h.default` 複製一份並重新命名為 `src/config.h`。
    *   在 `src/config.h` 中填入您的 WiFi SSID 與密碼。

4.  **編譯與上傳**:
    *   使用 PlatformIO 的 "Upload" 功能將程式碼上傳至您的 ESP32 開發板。

5.  **操作**:
    *   上傳成功後，打開序列埠監控視窗（Baud Rate: 115200），您將會看到 ESP32 連上 WiFi 後所分配到的 IP 位址。
    *   在同一區域網路下的手機或電腦瀏覽器中，輸入該 IP 位址。
    *   點擊網頁上的 "Start Scan" 按鈕，即可開始自動化掃描。掃描期間，圖表會即時更新。
