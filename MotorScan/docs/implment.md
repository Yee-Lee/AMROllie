## 1. 專案目的

本專案旨在建立一個馬達控制系統，該系統能夠：
本專案旨在建立一個基於 ESP32 的馬達特性掃描與控制系統。主要用於分析直流馬達的 PWM 與 RPM 響應特性，並透過 Web 介面進行即時監控與操作。

*   透過網頁介面進行遠端控制。
*   支援 PID 控制模式，以精確控制馬達轉速。
*   支援 PWM 直接控制模式，用於手動調整馬達輸出。
*   提供即時的馬達狀態監控，包括轉速和 PWM 輸出。
*   使用模擬馬達進行初步開發和測試，無需實際硬體。
## 2. 系統架構

## 2. 實作細節
### 2.1 核心類別

### 2.1 程式架構
*   **`Motor`**: 硬體抽象層。控制 TB6612FNG 等驅動板，讀取編碼器 (Encoder) 訊號以計算轉速。
*   **`MotorScanner`**: 自動化掃描邏輯。負責執行 PWM 掃描序列（例如從低 PWM 到高 PWM），用於建立馬達的特性曲線。
*   **`MotorWebUI`**: 使用者互動介面。基於 `ESPAsyncWebServer` 與 `AsyncTCP`，提供：
    *   **圖表 (Chart.js)**: 顯示 RPM 變化曲線。
    *   **終端機 (Terminal)**: 顯示詳細數據 (Timestamp, RPM, PWM)，支援大量數據滾動。
    *   **WebSocket**: 實現低延遲的雙向控制與數據傳輸。
*   **`config.h`**: 獨立的設定檔，用於儲存 WiFi SSID 與密碼等敏感資訊 (由 `config.h.default` 產生)。

本程式主要由以下幾個類別組成：
### 2.2 主要功能

*   **`Motor`**: 用於控制真實馬達的類別。它使用中斷來計算編碼器的脈衝，從而追蹤馬達的位置和轉速。
*   **`MotorSim`**: 用於模擬馬達行為的類別。它可以模擬馬達的響應時間、死區和飽和區。
*   **`PIDController`**: 用於實現 PID 控制算法的類別。它可以根據目標值和當前值計算輸出，並具有抗飽和機制。
*   **`MotorWebUI`**: 用於建立網頁使用者介面的類別。它使用 WiFi 連線來提供網頁介面，以控制和監控馬達。
1.  **馬達切換控制**: 支援雙馬達系統 (Left/Right)，可透過網頁介面動態切換當前控制與監控的馬達對象。
2.  **自動特性掃描**: 系統進入 `RUNNING` 狀態後，`MotorScanner` 會自動控制馬達進行測試流程。
3.  **即時監控**: 每 100ms 廣播一次 PWM 與 RPM 數據至網頁端，避免網路壅塞。
4.  **狀態管理**: 透過狀態機 (IDLE, RUNNING) 管理系統運作，確保測試流程的可控性。

### 2.2 主要功能
### 2.3 軟體流程

*   **網頁介面**: 使用者可以透過網頁介面設定目標 RPM、調整 PID 參數、切換控制模式和啟動/停止系統。
*   **PID 控制**: 系統使用 PID 控制器來自動調整馬達的 PWM 輸出，以達到目標轉速。
*   **PWM 直接控制**: 使用者可以直接設定 PWM 輸出值，從而手動控制馬達轉速。
*   **即時監控**: 系統會將馬達的當前轉速、目標轉速和 PWM 輸出等資料，即時顯示在網頁介面上。
1.  **初始化**: 
    *   設定 Serial (115200)。
    *   初始化左右馬達物件 (GPIO 設定)。
    *   連接 WiFi 並啟動 Web Server。
2.  **主迴圈 (`loop`)**:
    *   檢查 WebUI 是否有切換馬達請求，若有則更新 `currentMotor` 指標。
    *   **IDLE 狀態**: 等待網頁端的 "Start System" 指令。若收到指令，切換至 RUNNING。
    *   **RUNNING 狀態**:
        *   若 `MotorScanner` 為 IDLE，則啟動掃描 (`scanner.begin()`)。
        *   執行 `scanner.run()` 進行測試邏輯。
        *   定期 (100ms) 透過 WebSocket 廣播當前轉速與 PWM 值。
    *   若收到 "Stop System" 指令，切換回 IDLE 並重置掃描器。

## 3. 硬體配置 (ESP32)

1.  程式初始化 WiFi 連線、馬達物件和網頁介面。
2.  在主迴圈中，程式會定期更新馬達狀態、計算 PWM 值，並透過網頁介面廣播資料。
| 功能 | 左馬達 (Left) | 右馬達 (Right) | 備註 |
| :--- | :--- | :--- | :--- |
| **IN1** | GPIO 12 | GPIO 27 | 方向/速度控制 |
| **IN2** | GPIO 14 | GPIO 26 | 方向/速度控制 |
| **PWM** | GPIO 13 | GPIO 25 | PWM 輸出 |
| **Encoder A** | GPIO 18 | GPIO 19 | 編碼器 A 相 |
| **Encoder B** | GPIO 34 | GPIO 35 | 編碼器 B 相 |

*   **脈衝數 (PPR)**: 146
*   **減速比**: 20

## 4. Web 介面操作

*   **SELECT MOTOR**: 下拉選單選擇要測試的馬達 (LEFT / RIGHT)。測試進行中 (RUNNING) 會鎖定此選項。
*   **Start/Stop System**: 控制測試開始與結束。
*   **Terminal**: 顯示即時回傳的數據流，包含時間戳記、RPM 與 PWM 值 (Buffer 上限 500 筆)。
