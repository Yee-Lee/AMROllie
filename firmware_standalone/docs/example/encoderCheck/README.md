# Encoder Check (Encoder 脈衝測試)

本目錄 (`example/encoderCheck/`) 是一個獨立的測試專案，用於精準測量並驗證馬達轉動一圈所產生的 Encoder 脈衝數 (CPR, Counts Per Revolution)。

## 專案目標

為了讓機器人的 PID 控制器與里程計 (Odometry) 能準確運作，系統必須知道「輪子轉一圈會觸發多少次中斷」。
這支測試程式排除了主系統所有複雜的控制邏輯，純粹在底層綁定硬體中斷，並完全對齊主程式的 **2x Decoding (雙沿觸發)** 邏輯。透過這支程式測得的數據，可以直接填入系統設定中，確保轉速計算零誤差。

## 如何使用

1. **硬體接線與腳位確認**:
   請確認您的馬達 Encoder A/B 相已連接至 ESP32，並在 `src/config.h` 中設定了對應的腳位（例如 `MOTOR_L_ENC_A`, `MOTOR_L_ENC_B` 等）。

2. **編譯與上傳**:
   在 PlatformIO 中，將編譯環境切換為 **`env:encoder_check`**。
   編譯並將程式上傳至 ESP32 開發板。

3. **開始測試**:
   - 開啟序列埠監控視窗 (Serial Monitor)，Baud Rate 設定為 `115200`。
   - 畫面上會每 500ms 印出目前的計數值 (Count) 與原始腳位狀態 (Raw Pins)。
   - **用手動方式將左輪或右輪「剛好」旋轉一圈**。
   - 觀察畫面上增加 (或減少) 的 `Count` 數值，這就是該馬達的實際 CPR。

4. **更新設定**:
   將測量到的數值（例如 `294`）填入主專案 `src/config.h` 的 `MOTOR_CPR` 參數中：
   ```c
   #define MOTOR_CPR 294
   ```

5. **歸零與重複測試**:
   若想分次測量或重新計算，請在序列埠監控視窗的輸入框中輸入小寫 `r` (或大寫 `R`) 並送出，計數器即會歸零。

## 硬體除錯 (Troubleshooting)

如果您轉動輪子時，發現 `Count` 數值始終維持在 `0` 沒有變化，請觀察畫面後方的 `Raw Pins` 狀態：
- **正常情況**：慢慢轉動輪子時，A 與 B 相的數字會在 `0` 和 `1` 之間不斷跳動。
- **若數字死卡在 `1` 或 `0`**：
  - 檢查 Encoder 模組是否有妥善連接供電 (VCC / GND)。
  - 檢查杜邦線是否鬆脫或腳位定義是否對應。
  - **ESP32 腳位地雷**：若您剛好將腳位接在 ESP32 的 **34, 35, 36, 39**，這些純輸入腳位**不支援內部上拉電阻 (`INPUT_PULLUP`)**。若您的 Encoder 需要上拉電阻才能產生高電位，接在這裡將讀不到訊號。解法為：自行於電路外部加上 10k 上拉電阻，或是換到其他具備內部上拉功能的 GPIO (如 25, 26, 27, 32, 33 等)。
